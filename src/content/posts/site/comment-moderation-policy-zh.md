目前站点评论托管在 `Giscus + GitHub Discussions` 上。

这样做的原因很直接：第一版先把评论落到维护成本最低、审计路径最清楚的平台侧能力上，后续如果确实需要更细的私有规则、账号体系或更强的自动过滤，再考虑迁到自建后端。

## 欢迎的内容

- 针对文章内容的补充、纠错与追问
- 与主题直接相关的经验、案例与反例
- 基于事实和理由的不同意见
- 对排版、链接、引用或可读性问题的反馈

## 可能被处理的内容

以下内容不适合保留在评论区：

- 明显广告、导流、SEO 外链投放或批量灌水
- 钓鱼、骗取凭据、恶意下载、钱包助记词或私钥相关诱导
- 威胁、骚扰、人身攻击或故意刷屏
- 泄露他人隐私、身份信息或其他不应公开的数据
- 与文章主题无关、且只以引流或破坏讨论为目的的内容

## 自动处理的边界

自动化只处理非常窄的一类情况：

- 带有明显广告或诈骗特征的评论
- 带有明显恶意链接、凭据诱导或恶意软件分发特征的评论

命中这类规则后，评论可能会在 GitHub 侧被自动最小化，并保留平台可见的原因，例如 `spam` 或 `abuse`。

除此之外，自动化不会尝试替代人工判断。边界模糊、需要上下文解释、或者只是观点尖锐但仍在讨论范围内的评论，不会因为自动规则直接被处理。

## 如果发生误判

如果你的评论被最小化，但你认为这是误判，可以：

- 直接新开一个 GitHub issue，并附上对应 discussion 链接
- 简要说明评论原意，以及你认为被误判的原因

我会优先按公开规则复核，而不是让隐藏规则替代说明。
