# Strategy Template

Use this structure for implementation strategy output:

## Scope

- Affected modules or workflows
- What is in scope
- What is out of scope

## Key Path

1. First change
2. Follow-up change
3. Integration point

## Risks

- Main regression or integration risks
- Areas requiring special review

## Validation Plan

- Checks to run
- Behavior to verify
- Scenarios that must not regress

## Phased Order

1. Preparation
2. Implementation
3. Verification
