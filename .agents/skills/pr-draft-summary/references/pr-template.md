# PR Template

Use this structure for pull request summary output:

## Title

- `type(scope): summary`

## Summary

- What changed
- Why the change was necessary

## Scope

- Main implementation points
- Explicit out-of-scope follow-up if needed

## Validation

- Commands run
- Result
- Skipped checks with reason

## Issue Links

- Parent: `#...`
- Sub-issue: `#...`

## Risks and Follow-ups

- Residual risks
- Deferred work
